1 简介

Vimwiki 是一个个人知识管理工具， 基于vim，项目主页：vimwiki - Personal Wiki for Vim - Google Project Hosting

语法简单，操作容易，可生成html页面，我主要用它记录自己的学习笔记，也可作为写日记的工具。

2 基本的Wiki语法

2.1 标题

=一级标题=
==二级标题==
===三级标题===
以此类推。

2.2 文本语法

*粗体*  _斜体_  ~~删除线~~  `Some Code 代码`

注意这几个针对文本格式的标签，都要求左右留有空白。
请注意你的代码高亮，一般来说，在编辑器中有了相应的高亮，你用的wiki标签才生效。

2.3 blockquote

四个空格缩进的内容会被转成blockquote

2.4 pre标签

{{{内容}}} 由三个大括号括起来的内容，生成html时会被放到pre标签中，适合贴代码。

2.5 维基词条

WikiItem 大写开头的驼峰英文会被自动当作一个维基词条，并添加链接 [[Wiki Item]] 这是手动建立维基词条的方式 [[wiki item|description]] 输出HTML时显示description,链到wiki item http://ktmud.com/ 外部URL会被自动转成链接 [http://ktmud.com Ktmud] 带文字的外链 [images/hello.jpg] 输出<img src="images/hello.jpg"/> [[images/hello.jpg]] 输出图片，并链向图片地址

2.6 列表

* 无序列表 条目一
* 无序列表 条目二
 - 子列表 条目一
 - 子列表 条目二

# 有序列表 条目一
# 有序列表 条目二

* 和 - 是等价的，后面必须跟一个空格

2.7 注释

  • 注释代码行以%%开头
  • 注释代码段用<!-- 内容 -->

3 特殊占位符

在wiki条目中使用以下占位符，能对生成的HTML文件做一些特殊的处理

  • %toc 自动生成目录
  • %title name 指定HTML文档的title。如果不指定，title就是文件名
  • %template name 为该维基页面指定一个HTML生成模版
  • %nohtml 告诉vimwiki不为某条目生成HTML文件。

4 生成HTML

使用 :Vimwiki2HTML 可以为当前维基条目生成HTML文件；快捷键：<leader>wh 使用 :VimwikiAll2HTML 可以为所有条目生成HTML。

5 语法高亮

  • 使用插件SyntaxHighlighter
  • 在页面模板中添加scripts和css链接
  • 设置python语法高亮

    print(i)

7 个人配置及心得

7.1 我的vimrc配置

" vimwiki
let g:vimwiki_use_mouse = 1
let g:vimwiki_camel_case = 0
let g:vimwiki_html_header_numbering = 2
let wiki = {'path': '~/vimwiki/',
            \'path_html': '~/projects/chenxianmin.github.com/wiki/',
            \'nested_syntaxes': {'python': 'python'},
            \'template_path': '~/vimwiki/template/',
            \'template_default': 'def_template',
            \'template_ext': '.html'}
let blog = {'path': '~/projects/chenxianmin.github.com/_posts/',
            \'ext': '.textile'}
let g:vimwiki_list = [wiki, blog]

7.2 心得感悟

  1. 只要学会了基本语法就能开始记录了。之后多查阅vimwiki帮助文档，会有意外的发现。
  2. vimwiki更适合自己创作书写，将看到的知识点、技巧总结在内，将自己的想法记录；并不适合作为链接存储和网页摘录。
  3. 在使用vimwiki的过程中，对于熟悉html/css很有帮助，我的页面主要是模仿他人的源码自己修改而成的。
  4. 在每个条目的上方添加%toc可为页面生成目录，再修改css将目录美化，这对于浏览者有很好的体验。
  5. 生成页面时使用不同的页面模板，比如我的：需要语法高亮的页面我会使用带有语法高亮插件的模板。模板的使用方法参见帮助文档。注意%title%, %root_path%, %context%的使用。
  6. 可为每个条目的上方添加一个标题%title name,之后在模板中，%context%的上方添加一句<h1>%title%</h1>。
  7. 使用多个wiki项目。我另建了一个项目用于写博客。
  8. 用[[目录名/Item]]这个方法，可将条目放置于指定目录内，方便管理。
  9. 可以在vimrc中修改文件的后缀，比如.markdown, .textile等。
